# KUMA XP/Leveling UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the existing XP/leveling engine in the UI: an XP bar + level on the dashboard (both normal and Kuroshuna skins), "+EXP" on the battle victory screen, and a transient "network discovered" EXP toast on the home screen.

**Architecture:** The backend already computes progress (`kuma_core/progress.py`: 30 XP/level, `get_progress()` returns `xp_into_level` 0..29 + `xp_to_next`). Add those two to `/api/status`. Firmware parses them, draws a fill bar (`xp_into_level / 30`) next to the level, shows the fixed battle reward on victory, and detects a `network_count` increase between polls to flash a discovery toast.

**Tech Stack:** FastAPI (TDD via TestClient), C++/LovyanGFX (compile-verified). The Pi level-reset (disable creator-lock, start Shuna at 1) is a separate SSH step, not in this plan.

**How to run tests:** backend `python -m pytest tests/test_kuroshuna_api.py -v`; firmware `pio run -e t-deck`.

### Task 3: firmware — XP bar + level in drawHome (both skins)

**Files:** `firmware/tdeck-ui/src/kuma_ui.cpp`

READ `drawHome` first. Both the normal top bar (wordmark left, `Lv N` to its right, online dot far right) and the Kuroshuna mirrored bar (wordmark+`Lv N` right) currently show only the level. Add a small XP **fill bar** immediately to the LEFT of the `Lv N` text, filled by `xpIntoLevel / (xpIntoLevel + xpToNext)` (== `/30`). A helper keeps both skins consistent:

```cpp
// Draw a small XP bar [w x 6] at (x,y) filled by into/(into+toNext), then return
// the x just past it. Colors per skin (cyan/green normal, red kuro).
static int drawXpBar(lgfx::LovyanGFX* g, int x, int y, int w,
                     uint16_t into, uint16_t toNext, uint16_t fillCol, uint16_t frameCol) {
  int total = into + toNext; if (total <= 0) total = 1;
  g->drawRect(x, y, w, 6, frameCol);
  g->fillRect(x + 1, y + 1, (w - 2) * (int)into / total, 4, fillCol);
  return x + w;
}
```

### Task 5: firmware — network-discovery EXP toast on home

**Files:** `firmware/tdeck-ui/src/main.cpp`, `firmware/tdeck-ui/src/kuma_ui.cpp`/`.h`

When `g_status.networkCount` increases between status polls, flash a small bottom-of-screen toast on the Home screen for ~3s: `+1 EXP  new network`.

- [ ] **Step 1 (kuma_ui):** add a tiny API the home can call to set a transient toast. Simplest: a module-level `static uint32_t g_toastUntil = 0; static String g_toastMsg;` in kuma_ui's anonymous namespace + a public `void kuma_ui::toast(const String& msg, uint32_t ms = 3000)` that sets `g_toastUntil = millis() + ms`. In `drawHome`, after the stat bar, if `millis() < g_toastUntil`, draw `g_toastMsg` centered at the very bottom (y~232) in a readable color over a short dark strip. (drawHome runs ~10fps so it animates out.)

Declare in `kuma_ui.h`: `void toast(const String& msg, uint32_t ms = 3000);`

- [ ] **Step 2 (main.cpp):** in the status-poll block, after a good poll updates `g_status`, compare to the previous count:
```cpp
  static uint32_t s_lastNetCount = 0;
  if (s_lastNetCount && tmp.networkCount > s_lastNetCount)
    kuma_ui::toast(String("+1 EXP  new network (") + tmp.networkCount + ")");
  s_lastNetCount = tmp.networkCount;
```
(Guard `s_lastNetCount` nonzero so the first poll doesn't toast the whole backlog.)

- [ ] **Step 3:** `pio run -e t-deck` → SUCCESS + flash %.
- [ ] **Step 4: Commit** `feat(fw): home toast '+1 EXP new network' on discovery`
